Commit Changes

Changes are saved in GitHub by committing. This overviews how to edit documentation on GitHub and then commit the changes to the repository on your personal account.

Prerequisites​

  • A copy of the repository is suggested to review and edit documentation.
  • Check the language used for the documentation to be sure you are familar with it.

Steps​

  1. Select the repository from Your repositories.

  2. Be sure you are on the main page located in the Code tab.

  3. Click on the document you'd like to revise.

  4. Click the edit button on the top-right corner. Usually it's a pencil icon, but could be different dependent upon the documentation framework. This should open the document in the text editor.

  5. After you have entered the changes, scroll down to the Commit Changes section. Enter a descriptive title which helps the administrators know what the changes address. Below the title is a description box for additional details.

  6. Select Create a new branch for this commit and start a pull request. Click Propose changes.

  7. You will be directed to the Comparing changes page which lists all of the edits. The edits are categorized as an addition or deletion.

Results​

The repository in your account now has a new branch containing the edits.
